# `dAI+정리진행` vs `inni-platform` 구현 방식 및 분석 철학 차이

두 프로젝트는 기술적 기반(Python, Streamlit, DSPy)을 공유하지만, 분석의 **공간적 스케일(Scale)**과 **데이터 처리의 깊이**에서 본질적인 지향점이 다릅니다.

## 1. 공간적 스케일 및 분석 대상 (Scale & Scope)
- **dAI+정리진행 (Architecture Scale)**: 
  - 특정 필지(Site) 내에서의 **건축물** 생산에 집중합니다. 
  - 용적률, 건폐율, 층고, 공사비 등 물리적 수치와 건축 법규의 정밀한 검증이 핵심입니다.
- **inni-platform (Urban Scale)**: 
  - 개별 건물을 넘어 **도시(City) 및 권역(Region)** 단위의 분석을 지향합니다.
  - 지도(Map) 기반의 공간 데이터, 인구 통계, 광역 교통망, 정책 변화 등 훨씬 방대하고 파편화된 데이터를 통합하여 **지속 가능한 도시 전략**을 도출하는 것이 핵심입니다.

## 2. 주요 기능 구현 방식 차이

| 구분 | dAI+정리진행 (건축 중심) | inni-platform (도시 중심) |
| :--- | :--- | :--- |
| **문서 업로드** | PDF 중심 (설계도서, 사업계획서) | 멀티 포맷 (Excel 통계, CSV 데이터, 보고서 등) |
| **파싱 전략** | 텍스트 추출의 정확도와 고속화 강조 | 텍스트 품질 점수화 및 Gemini 비전 기반 Native 파싱 (도표/지도 이해) |
| **청킹/RAG** | ChromaDB 등 벡터 DB 기반의 영구 인덱싱 | 인메모리 기반의 유연한 검색 및 Overlap을 통한 문맥 보존 |
| **AI 추론** | Claude 기반의 정교한 ReAct 체인 | Gemini 기반의 고속 처리 및 Structured Output (JSON) |
| **프롬프트** | 건축 전문 도메인이 반영된 고정형 Signature | RISEN 구조를 통한 동적 블록 생성 및 사용자 피드백 반영 |

## 🚀 inni-platform (도시 단위) 고도화를 위한 dAI+ 기반 제안

도시 단위 분석은 개별 건축물 분석보다 **데이터의 상호 연관성(Synthesis)**과 **논리적 신뢰성(Reliability)**이 훨씬 중요합니다. 이를 위해 dAI+의 고도화된 모듈 중 다음 요소들을 도시 분석 맥락으로 재해석하여 도입할 것을 제안합니다.

### 1. 하이브리드 검색 및 지능형 재랭킹 (For 복합 데이터 합성)
*   **도시 분석 환경**: 도시 계획 보고서, 통계 자료, 기사 등 출처가 다양하여 단순 유사도 검색으로는 핵심 문맥을 놓치기 쉽습니다.
*   **도입 제안**: dAI+의 `RetrieveMix`를 도입하여 **키워드(고유명사, 지명)와 의미(트렌드, 정책 방향)**를 동시에 검색하고, 추출된 정보들을 **Claude를 이용해 재랭킹**함으로써 도시 분석의 논리적 재료를 더 정교하게 수집할 수 있습니다.

### 2. 출처 검증 및 환각 감지 (For 정책 결정의 신뢰성)
*   **도시 분석 환경**: 도시 단위 의사결정은 사회적 영향력이 크므로, AI의 제안이 실제 어떤 법령이나 통계에 기반했는지 투명하게 드러나야 합니다.
*   **도입 제안**: dAI+의 `Source Verification` 시스템을 이식하여, 모든 분석 문장에 대해 **원본 데이터(출처) 매핑 및 환각 위험 점수**를 제공합니다. 이는 '도시 전략'의 근거를 사용자가 즉시 검토할 수 있게 합니다.

### 3. 광역 지표 자동 추출 및 논리 검증 (For 데이터 정합성)
*   **도시 분석 환경**: 인구 밀도, 유동 인구, 용도지역 비율 등 도시 단위 지표는 단위가 다양하고 수치가 큽니다.
*   **도입 제안**: dAI+의 `Building Metrics` 추출 로직을 **'Urban Indicators Extractor'**로 확장합니다. 단순히 텍스트를 읽는 것이 아니라, 추출된 수치들 간의 산술적/논리적 정합성(예: 거주 인구수와 가구수 간의 상관관계 등)을 자동으로 검증하는 파이프라인을 구축합니다.

### 4. 심층 추론 파이프라인 (Requirement-Reasoning-Strategy)
*   **도시 분석 환경**: "이 지역은 활력이 부족하다"는 현상 분석에서 바로 "아트센터를 짓자"는 결론으로 비약하기 쉽습니다.
*   **도입 제안**: dAI+의 `AdvancedAnalysisPipeline` 구조를 적용하여, **[도시 현황 파악] -> [원인 및 논리 추론] -> [공간 전략 도출]**로 이어지는 다단계 DSPy 체인을 강화합니다. 이를 통해 '도시 분석' 자체의 깊이와 설득력을 비약적으로 높일 수 있습니다.
